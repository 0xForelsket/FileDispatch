## Design Philosophy

### Core Principles

1. **Invisible When Working**
   - Runs silently in background
   - No unnecessary interruptions
   - Tray icon for quick status check

2. **Visible When Needed**
   - Easy access to activity logs
   - Clear preview of rule effects
   - Undo when mistakes happen

3. **Progressive Disclosure**
   - Simple rules are simple to create
   - Advanced features don't clutter basics
   - Complexity revealed on demand

4. **Confidence Through Feedback**
   - Preview before enabling rules
   - Clear success/failure indicators
   - Explanations for why rules match or don't

## Interaction Patterns

### Adding a Rule

```
1. User clicks "+ Add Rule"
2. Rule editor opens with empty state
3. User enters rule name
4. User adds conditions (defaults to "Name contains")
5. User adds actions (defaults to "Move")
6. User clicks "Preview" to test
7. Preview shows which files would match
8. User clicks "Save Rule"
9. Rule added to list, enabled by default
10. Toast notification confirms creation
```

### Editing a Rule

```
1. User clicks rule item or selects "Edit" from menu
2. Rule editor opens with current values
3. User modifies conditions/actions
4. User can preview changes
5. User clicks "Save Rule"
6. Changes saved, rule continues running
7. Toast notification confirms update
```

### Drag-to-Reorder Rules

```
1. User hovers over rule item
2. Drag handle appears on left
3. User drags rule up or down
4. Drop indicator shows insertion point
5. User releases
6. Rule order updated
7. New order persisted immediately
```

### Preview Mode

```
1. User clicks "Preview" in rule editor
2. Panel slides open beside editor
3. All files in watched folder evaluated
4. Results grouped by match/no-match
5. Each file shows condition breakdown
6. Matched files show planned actions
7. User can click file to expand details
8. Panel closes when editor closes
```

## Accessibility

### Keyboard Navigation

| Key | Action |
|-----|--------|
| `Tab` | Move focus forward |
| `Shift+Tab` | Move focus backward |
| `Enter` | Activate focused element |
| `Escape` | Close dialog/cancel |
| `Arrow Up/Down` | Navigate lists |
| `Space` | Toggle checkboxes/switches |

### Focus Indicators

- Visible focus ring on all interactive elements
- 2px solid primary color
- Offset to not overlap content

### Screen Reader Support

- Semantic HTML elements
- ARIA labels for icons
- Live regions for dynamic content
- Meaningful link/button text

### Color Contrast

- Text: minimum 4.5:1 contrast ratio
- Large text: minimum 3:1 contrast ratio
- UI components: minimum 3:1 against background

## Animation Guidelines

### Principles

- Subtle and purposeful
- Fast (150-300ms)
- Ease-out for entrances
- Ease-in for exits
- Reduce motion preference respected

### Standard Animations

```css
/* Fade in */
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

/* Slide up */
@keyframes slideUp {
  from { transform: translateY(10px); opacity: 0; }
  to { transform: translateY(0); opacity: 1; }
}

/* Scale in */
@keyframes scaleIn {
  from { transform: scale(0.95); opacity: 0; }
  to { transform: scale(1); opacity: 1; }
}
```

### Usage

| Element | Animation | Duration |
|---------|-----------|----------|
| Dialog open | Scale in | 200ms |
| Toast appear | Slide up | 200ms |
| List item add | Fade in + slide | 150ms |
| Button hover | Background color | 150ms |

## Platform Considerations

### Linux

- Use system title bar (Tauri default)
- Respect GTK/Qt theme where possible
- Use system file dialogs
- Tray icon in AppIndicator format

### Windows

- Can use custom title bar if desired
- Follow Windows 11 design language
- Use system file dialogs
- Tray icon in system tray

### Future: macOS

- Use native title bar
- Support native menu bar
- Follow macOS design conventions
- Menu bar icon instead of tray
